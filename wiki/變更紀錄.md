# 2026-03-28 更新內容

### 🔒 Race Condition 修復 (Concurrency Safety)
- **`todaySaleRecord.js` `deleteTodaySaleRecord()`：** 原本完全沒有 LockService 保護，刪除標記與退點操作非原子性。現已包覆 `ScriptLock`，並改用 `_addPointsUnsafe` 避免雙重拿鎖。
- **`todaySaleRecord.js` `closeAccountBox()`：** 原本使用 `DocumentLock`，與 `apiCheckout` 等 API 使用的 `ScriptLock` 不互斥，關帳搬移資料時結帳可同時寫入導致資料遺漏或被清除。已改為 `ScriptLock`，並將逐行 `appendRow` 改為批次 `setValues` 提升效能與安全性。
- **`services.js` `結帳()`：** 原本點數更新與銷售紀錄寫入使用兩把不同的鎖，若點數扣除成功但寫入銷售紀錄失敗會造成資料不一致。已合併為單一 `ScriptLock` 包覆，並改用 `_addPointsUnsafe`。
- **`addLottery.js` `福袋入檔()`：** 移除鎖外的死代碼 `finalId = getMaxValueOfColumnA() + 1`（該值已在鎖內被 `actualFinalId` 覆寫）。

### 🎨 UI/UX 統一刷新按鈕 & 分頁持久化
- **統一 `RefreshButton` 元件：** 新增 `RefreshButton.tsx`，全分頁共用統一的刷新資料按鈕（indigo→violet 漸層 + loading 脈衝動畫 + 旋轉圖示 + 文字切換「讀取中…」）。已套用至：當日銷售、會員資料、歷史銷售紀錄、福袋獎項庫、貨品資料庫、盲盒資料庫。
- **瀏覽器重整保持分頁：** `activeTab` 與 `branch` 改用 `useStickyState` 持久化至 localStorage，F5 重整後自動留在當前分頁與門市並重新載入資料。
- **銷售紀錄 F5 強制重抓：** 初次載入時跳過 localStorage cache 直接重抓最新 1000 筆；切換 tab 再回來則使用 cache。
- **移除關帳 summary row：** `apiCloseDay` 不再產生 `【系統結帳紀錄】` 的 summary 行。
- **修復浮點精度顯示：** 在 `gasPost()` 加入 `roundFields()` 遞迴處理，對 `points`、`pointDelta`、`amount`、`cash` 等數值欄位統一 `Math.round()`，消除 `110.00000000000001` 這類 IEEE 754 誤差。
- **銷售紀錄顯示會員名字：** 歷史銷售紀錄頁面的交易 header 現在會根據電話號碼查找並顯示會員姓名。

### 📋 後端操作紀錄 (Audit Logging)
- **開關帳紀錄 sheet（12 欄）：** 關帳成功後自動寫入一列紀錄，包含：關帳時間、門市、操作人員 email、開櫃準備金、應收/實收現金、現金差異、信用卡總額、匯款總額、本日營業額、交易筆數、備註。首次寫入時自動補標題列。
- **廢套紀錄 sheet（11 欄）：** 作廢獎項庫後自動寫入一列紀錄，包含：作廢時間、門市、操作人員 email、套號、套名、單抽價、獎項數、總抽數、獎項明細（分號分隔）、是否點數套、原始建套日期。
- **後端浮點修復：** GAS 後端在產生廢套明細時，對 `points` 欄位加入 `Math.round()` 避免浮點誤差寫入紀錄。

# 2026-03-19 更新內容

### 🎨 介面與體驗優化 (UI/UX Improvements)
- **結帳頁面版面微調：** 大幅壓縮「福袋清單」與「直購商品清單」的次要資訊（如單號、數量、單價）欄位寬度，並將資源留給「大套名稱」和「商品名稱」，優先突顯最重要的資訊。
- **編輯狀態視覺化：** 針對所有輸入框進行重構，清楚區分「可編輯」與「唯讀」欄位。唯讀欄位以褪色且無外框的方式呈現，降低視覺干擾。
- **貨品資料頁面大翻新：** 捨棄過時的灰色表頭與死板的警告色塊。導入現代化圓角搜尋卡片、整合式搜尋圖示，並使用柔和的徽章取代突兀的警告標語，優化資訊層次。
- **響應式佈局 (RWD)：** 解除結帳頁面的最大寬度限制，讓大螢幕能完整展開。同時為表格加入防擠壓設計 (`min-w-[800px]`)，確保手機版滑動體驗順暢。

### ⚡ 結帳操作與防呆 (Checkout & Safety)
- **結帳資料防丟失機制：** 結帳畫面所有輸入狀態（顧客資料、福袋明細、商品清單、收款金額）皆透過 `useStickyState` 寫入本機快取。即使不小心按到 F5 重新整理，資料也原封不動保留。
- **一鍵重置結帳：** 於結帳按鈕旁新增「全新結帳」功能，防呆確認後可瞬間淨空畫面上所有快取資料，方便迎接下一位顧客。
- **會員輸入輔助：** 會員電話輸入支援自動完成，點選快取清單後會順帶在欄位下方顯示該會員「姓名」以供雙重核對。
- **特定獎項 (Z) 邏輯處理：** 當福袋獎項輸入大寫或小寫 `Z` 時，名稱自動帶入「非GK」，點數預設為 `1`。客戶選「帶走」時無點數，選「點數」則依抽數計算總點數。
- **GK 直購鎖定防呆：** 掃入貨品資料庫的 ID 時，系統會自動判斷該商品是否屬於 GK 類別。 若為 GK，結帳付款清單將**強制隱藏「現金」選項**（包含防呆自動切換為點數），避免誤用現金結帳。盲盒則正常開放手動售價與現金付款。

### 📜 歷史紀錄與相容性強化 (History & Compatibility)
- **消費紀錄資訊全透明：** 針對福袋明細，於單號下方明確標示顧客選擇「帶走」或「點數」；若是點數兌換亦會顯示增加/扣除之額外點數，清楚無比。
- **直購扣點標籤：** 當直購商品（含隨便扣 88888、99999）付款方式為點數兌換時，現在明細會追加紫色的 `點數兌換` 與結算 `扣 XX 點` 的醒目標籤，一眼看穿金流與點流。
- **舊版資料相容重生 (`web_api.js`)：** 為了找回過去舊版結帳流程中消失的點數扣抵數據，後端全面重寫了解析核心，讓成千上萬筆歷史紀錄「自動轉譯」成新版框架能讀懂的乾淨格式。
- **福袋獎項點數顯示優化：** 解決 Google 試算表帶來的浮點數小數點過長問題（例如 `110.00000001`），自動進位並隱藏多餘尾數。
- **資料檢索防呆：** 修復在搜尋列輸入特定會員姓名或字元時導致網頁崩潰 (Crash) 的型別錯誤 Bug。
