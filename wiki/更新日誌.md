## 2026-03-28 (v2.4)
### 🔒 Race Condition 修正
- **`apiCheckout` 點數寫入重構**：移除依賴前端傳入 `currentPoints` 的 stale data 模式，改用 `addMemberPointsByPhone` 在鎖內讀取 DB 即時點數進行相對值加減（`pointsDelta = totalCheckPoint - costToPayPoint`）
- **`addMemberPointsByPhone` 強化**：新增點數不足保護（回傳 `-2`），並修正數值轉型 `Number(data[i][6]) || 0`
- **前端 payload 精簡**：`useCheckout.ts` 移除 checkout API payload 中多餘的 `customer.currentPoints` 欄位

### 🛡️ 安全強化
- **JWT iss 驗證**：`verifyGoogleIdToken_` 新增 `payload.iss` 檢查，僅接受 `accounts.google.com` 或 `https://accounts.google.com` 簽發的 token
- **速率限制**：`apiMemberLogin` 使用 `CacheService` 實作同一電話 60 秒內最多 5 次登入嘗試，防止暴力破解
- **deleteRow 排序保護**：`apiDeleteDailySales` 和 `apiDeletePrizeLibrary` 的行號陣列新增顯式 `sort((a, b) => b - a)`，防止反向迭代時行號偏移

## 2026-03-25 (v2.1)
### ⚡ 效能優化 (Performance)
- **會員消費紀錄查詢 v3**：重寫 `apiGetMemberSalesRecords`，從逐行 getRange (N+1 problem) 改為 Column-A scan + batch read，API 呼叫次數從 N+1 降至 2 次
- **CacheService 快取**：會員消費紀錄加入 5 分鐘伺服端快取，重複查詢同一會員歸零延遲
- **前端並行查詢**：`fetchMemberHistory` 改用 `Promise.all` 並行呼叫 `getMember` + `getMemberSalesRecords`

### 🔒 安全性 (Security)
- **API Key 驗證**：GAS `doPost` 入口加入 `PropertiesService` API Key 比對，阻擋未授權存取
- **環境變數分離**：GAS URL 與 API Key 從 hardcode 遷移至 `.env` 環境變數（Vite `import.meta.env`）
- **`.gitignore` 排除**：`.env` / `.env.*` 不再被版控追蹤

### 🛡️ 並發安全 (Concurrency)
- **LockService 互斥鎖**：`apiCheckout`、`apiCloseDay`、`apiDeleteDailySales` 三個寫入函數加入 `LockService.getScriptLock().waitLock(30000)`，防止兩間門市同時操作導致資料覆蓋
- **異常安全釋鎖**：所有 Lock 區塊均使用 `try/finally { lock.releaseLock() }` 確保鎖必定釋放

### 🎨 UI/UX 改善
- **作廢訂單確認 Modal**：顯示客戶電話、結帳時間、實收金額、商品數，避免作廢錯誤訂單
- **分頁元件重寫**：頁碼按鈕 + 省略號 + 首尾頁跳轉 + active 高亮 + 總筆數顯示（銷售紀錄 & 當日銷售共用）
- **會員載入自動重試**：`fetchMembers` API 失敗時自動重試（最多 2 次，間隔 2 秒），並以 banner 通知使用者

### 📦 新功能：開套 (Create Set)
- **獎項庫「開套」按鈕**：5 步驟互動 Modal — ① 輸入貨號自動查詢名稱 ② 選擇抽數方案 (20/40/80/100/120) ③ 系統自動算出建議價格 ④ 店員輸入實際單抽價格 ⑤ 確認開套
- **防呆價格範圍**：實際單抽價格須在建議價 -8% 到 +50% 內（可在 `constants` 調整 `minPriceRatio`/`maxPriceRatio`）
- **開套成功提示**：彈出成功視窗顯示套號與套名，關閉後自動滾動至新建的套
- **商業邏輯集中管理**：抽數方案、價格公式乘數、折扣範圍統一由 `CREATE_SET_CONFIG` 管理
- **GAS 後端**：新增 `apiCreateSet`（LockService 保護套號唯一）、`apiGetStockItemByNo`（根據貨號查名稱與建議點數）
