# OneSoul POS 系統更新日誌

## 2026-04-02 (v2.9)
### 🔒 核心資安升級 (OWASP Security Fixes)
- **JWT 數位簽章驗證**：修補重大身分驗證漏洞 (OWASP A07)，`verifyGoogleIdToken_` 改採呼叫 Google 官方 `tokeninfo` 端點驗證密碼學簽章，防堵權限偽造。
- **防止試算表注入 (Formula Injection)**：修補資料隱碼漏洞 (OWASP A03)，實作 `sanitizeForSheet` 攔截所有 `=, +, -, @` 開頭的危險輸入，轉為安全純文字寫入 Google Sheets。

### ⚡ 後端效能與體驗優化 (Performance & UX)
- **廢套效能巨幅提升**：優化 `apiDeletePrizeLibrary` 邏輯，縮減高達 80% 的不必要資料讀取量。改進原先耗時的逐筆 `deleteRow()` 操作，重構成合併連續行號後的單次 `deleteRows(start, count)` 批次刪除，使整體執行時間從原本 10~15 秒顯著縮短至 3~5 秒。
- **JWT 快取機制**：為減輕 `tokeninfo` 端點驗證對每次 API 呼叫的額外 HTTP 開銷，導入 `CacheService` 對已驗證的 Token 提供 45 分鐘有效性快取，兼顧安全性與系統反應速度。
- **ScrollToTop 懸浮元件**：於前端全局加入右下角「回到頂部」的導覽輔助功能，便於瀏覽長篇幅商品與銷售清單。
- **值班人員整合通知**：強化 LINE 的溝通上下文，於「每日關帳通知」及「遲到打卡通知」的訊息中自動帶入並標示當日「值班人員」。

## 2026-04-02 (v2.8)
### 💬 LINE Messaging API 整合 (LINE Notification System)
- **Sheet-based Channel 制**：通知目標從 `ScriptProperties` 改為讀取 Google Sheet「LINE通知設定」分頁，支援多 channel（`all`、`admin`、`竹北`、`金山` 等）與多目標推送。每個 channel 可設定多個 group 或 user。
- **LINE Push API**：新增 `_linePush()` 底層函式，透過 LINE Messaging API 向指定 `groupId` 或 `userId` 推送文字訊息。
- **LINE Reply API**：新增 `_lineReply()` 函式處理 Webhook 指令回覆（免費不佔 Push 額度）。
- **ID 指令查詢**：使用者在 LINE 群組輸入「ID」（大寫），bot 自動回覆該群組的 `groupId` 或個人 `userId`，方便設定 Sheet。
- **Webhook 處理**：`doPost()` 新增 LINE Webhook 事件偵測，自動處理 `message` 類型的 ID 指令。
- **Channel 快取**：`sendNotify()` 使用 `CacheService` 快取 Sheet 設定（300 秒 TTL），減少 Sheet 讀取次數。
- **向下相容**：保留 `sendNotification()` 和 `sendAdminNotification()` 舊函式，確保現有業務邏輯不中斷。
