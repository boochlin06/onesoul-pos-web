# OneSoul POS 系統更新日誌

## 2026-03-19
### ✨ 新功能與優化 (Features & UI/UX)
- **結帳資料防丟失機制**：引入 Local Storage 狀態持久化，重新整理網頁不再遺失輸入到一半的結帳資料，並提供一鍵「全新結帳」重置按鈕。
- **結帳頁版面與資料層級最佳化**：縮減次要欄位以放大商品/套名顯示空間，並透過樣式明確重構區分「可編輯」與「唯讀」的表單狀態。支援全站響應式設計 (RWD)。
- **福袋與商品防呆邏輯**：當獎項輸入 `z/Z` 時自動設定為「非GK」與 `1` 點計算規則；同步新增單購 GK 阻擋現金付款的防護機制。
- **貨品資料頁 UI 大翻新**：重新設計搜尋區塊與列表表頭，汰換復古樣式並以柔和徽章 (Badges) 取代生硬的警告方塊。
- **會員輸入輔助**：優化結帳尋找會員的自動完成功能，選取後可立即預覽顧客姓名以二次核對。
- **當日銷售財務統計 (Daily Sales Financials)**：新增每日銷售視圖的財務統計數據，包含總營收、收取現金、信用卡交易額、匯款金額以及點數發放與折抵統計，提供一目瞭然的財務表現。
- **22 項銷售欄位扁平化顯示**：優化銷售紀錄的資料結構，完整呈現 22 項欄位資訊。
- **銷售紀錄快取機制**：加入 sales record caching，提升讀取速度與效能。
- **支援完整會員欄位**：補齊缺失的會員資料欄位，提升會員查詢與顯示的完整性。
- **消費紀錄資訊全透明與後端轉譯**：重寫後端資料解析核心 (`web_api.js`) 實現歷史相容性，找回失傳的扣減點數紀錄。會員歷史消費列表現會清楚標記福袋與商品「帶走」或「點數兌換」細節與額度。
- **獎項點數防呆修正**：修正搜尋特定歷史字元引發的崩潰情形，並修正福袋獎項小數點過長的問題。

## 2026-03-18
### 🚀 重大更新 (Major Update)
- **POS Web 前端現代化重構 (Frontend Migration)**：將原本基於 Google Sheets 的輸入介面，全面遷移至基於 React + Vite 開發的精美網頁應用程式 (SPA)。
- **Tailwind CSS 導入**：使用 Tailwind CSS 建立響應式、現代化且直覺的 UI 介面，優化結帳與購物車體驗。
- **API 與 GAS 後端整合**：前端無縫介接既有的 Google Apps Script (GAS) 後端，持續利用 Google Sheets 作為核心資料庫。
