## 2026-03-25 (v2.1)
### ⚡ 效能優化 (Performance)
- **會員消費紀錄查詢 v3**：重寫 `apiGetMemberSalesRecords`，從逐行 getRange (N+1 problem) 改為 Column-A scan + batch read，API 呼叫次數從 N+1 降至 2 次
- **CacheService 快取**：會員消費紀錄加入 5 分鐘伺服端快取，重複查詢同一會員歸零延遲
- **前端並行查詢**：`fetchMemberHistory` 改用 `Promise.all` 並行呼叫 `getMember` + `getMemberSalesRecords`

### 🔒 安全性 (Security)
- **API Key 驗證**：GAS `doPost` 入口加入 `PropertiesService` API Key 比對，阻擋未授權存取
- **環境變數分離**：GAS URL 與 API Key 從 hardcode 遷移至 `.env` 環境變數（Vite `import.meta.env`）
- **`.gitignore` 排除**：`.env` / `.env.*` 不再被版控追蹤

### 🛡️ 並發安全 (Concurrency)
- **LockService 互斥鎖**：`apiCheckout`、`apiCloseDay`、`apiDeleteDailySales` 三個寫入函數加入 `LockService.getScriptLock().waitLock(30000)`，防止兩間門市同時操作導致資料覆蓋
- **異常安全釋鎖**：所有 Lock 區塊均使用 `try/finally { lock.releaseLock() }` 確保鎖必定釋放

### 🎨 UI/UX 改善
- **作廢訂單確認 Modal**：顯示客戶電話、結帳時間、實收金額、商品數，避免作廢錯誤訂單
- **分頁元件重寫**：頁碼按鈕 + 省略號 + 首尾頁跳轉 + active 高亮 + 總筆數顯示（銷售紀錄 & 當日銷售共用）
- **會員載入自動重試**：`fetchMembers` API 失敗時自動重試（最多 2 次，間隔 2 秒），並以 banner 通知使用者

### 📦 新功能：開套 (Create Set)
- **獎項庫「開套」按鈕**：5 步驟互動 Modal — ① 輸入貨號自動查詢名稱 ② 選擇抽數方案 (20/40/80/100/120) ③ 系統自動算出建議價格 ④ 店員輸入實際單抽價格 ⑤ 確認開套
- **防呆價格範圍**：實際單抽價格須在建議價 -8% 到 +50% 內（可在 `constants` 調整 `minPriceRatio`/`maxPriceRatio`）
- **開套成功提示**：彈出成功視窗顯示套號與套名，關閉後自動滾動至新建的套
- **商業邏輯集中管理**：抽數方案、價格公式乘數、折扣範圍統一由 `CREATE_SET_CONFIG` 管理
- **GAS 後端**：新增 `apiCreateSet`（LockService 保護套號唯一）、`apiGetStockItemByNo`（根據貨號查名稱與建議點數）

## 2026-03-25 (v2)
### 🏗️ 架構重構 (Architecture Refactoring)
- **God Hook 消滅**：將 2,836 行的 `useAppData` 拆分為 6 個 domain hooks：
  - `useBanner` (26行) — 通知 banner + timer cleanup
  - `useMembers` + `useMemberHistory` (53行) — 會員 CRUD + 消費歷史
  - `usePrizes` (53行) — 獎品庫 + 作廢邏輯
  - `useSales` (149行) — 當日銷售 + 關帳 + 歷史銷售(含cache)
  - `useInventory` (33行) — 庫存 + 盲盒
  - `useCheckout` (130行) — 結帳業務邏輯（委派純函數）
- **純函數業務邏輯提取** (`logic/checkout.ts`)：
  - `calcSummary` — 金額/點數計算
  - `applyLotteryUpdate` — 福袋自動帶入獎項庫
  - `applyMerchUpdate` — 直購商品自動帶入庫存/盲盒
  - `filterMembers` — 會員 autocomplete 過濾
  - `validateCheckout` — 9 道結帳防呆驗證
- **CheckoutView Props 精簡**：從 20+ props 降至 9 個（自消費 `useCheckout`）
- **App.tsx 瘦身**：從 ~2,800 行降至 137 行，僅負責 layout 和 routing

### 🧪 單元測試 (Unit Tests)
- 新增 **38 個純函數測試**（Vitest），涵蓋：
  - 金額與點數計算 (8)
  - 福袋自動帶入 (8)
  - 直購商品自動帶入 (8)
  - 會員搜尋過濾 (5)
  - 結帳防呆驗證 (9)

### 🐛 修復 (Bug Fixes)

- 修正 `useEffect` dependency array 不完整的警告

## 2026-03-19
### ✨ 新功能與優化 (Features & UI/UX)
- **多重視圖優化 (Multi-View UI Modernization)**：
  - **當日銷售**：重構結帳明細渲染邏輯，引進「精簡對帳 / 經典表格」雙模式切換。新模式大幅收斂資訊，專注於商品、抽數、獎項、金流，提升店員快速審核體驗。
  - **會員資料**：移除擁擠的 8 欄表格並更換為「聯絡人名片卡」佈局，自動擷取字首頭像並放大顯示點數餘額；修正非字串資料導致的白屏錯誤。
  - **貨品資料**：更新為「卡片看板」，加入缺貨/低庫存提示；空名稱與無效商品自動隱藏，新增依據貨號遞減排序機制。
  - **盲盒資料庫**：重構為電商風格商品雙欄網格展台，統一引進「卡片陣列 / 經典表格」按鈕且具備使用者偏好自動記憶 (`useStickyState`) 功能。
- **結帳資料防丟失機制**：引入 Local Storage 狀態持久化，重新整理網頁不再遺失輸入到一半的結帳資料，並提供一鍵「全新結帳」重置按鈕。
- **結帳頁版面與資料層級最佳化**：縮減次要欄位以放大商品/套名顯示空間，並透過樣式明確重構區分「可編輯」與「唯讀」的表單狀態。支援全站響應式設計 (RWD)。
- **福袋與商品防呆邏輯**：當獎項輸入 `z/Z` 時自動設定為「非GK」與 `1` 點計算規則；同步新增單購 GK 阻擋現金付款的防護機制。
- **貨品資料頁 UI 大翻新**：重新設計搜尋區塊與列表表頭，汰換復古樣式並以柔和徽章 (Badges) 取代生硬的警告方塊。
- **會員輸入輔助**：優化結帳尋找會員的自動完成功能，選取後可立即預覽顧客姓名以二次核對。
- **當日銷售財務統計 (Daily Sales Financials)**：新增每日銷售視圖的財務統計數據，包含總營收、收取現金、信用卡交易額、匯款金額以及點數發放與折抵統計，提供一目瞭然的財務表現。
- **22 項銷售欄位扁平化顯示**：優化銷售紀錄的資料結構，完整呈現 22 項欄位資訊。
- **銷售紀錄快取機制**：加入 sales record caching，提升讀取速度與效能。
- **支援完整會員欄位**：補齊缺失的會員資料欄位，提升會員查詢與顯示的完整性。
- **消費紀錄資訊全透明與後端轉譯**：重寫後端資料解析核心 (`web_api.js`) 實現歷史相容性，找回失傳的扣減點數紀錄。會員歷史消費列表現會清楚標記福袋與商品「帶走」或「點數兌換」細節與額度。
- **獎項點數防呆修正**：修正搜尋特定歷史字元引發的崩潰情形，並修正福袋獎項小數點過長的問題。
