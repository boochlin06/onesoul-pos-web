# POS UI 測試流程

> 測試網址：`http://localhost:5174/onesoul-pos-web/#/`
> 正式環境：`https://boochlin06.github.io/onesoul-pos-web/`

---

## 0. 前置準備

// turbo
1. 確認 dev server 正在運行：`cd web && npm run dev`
2. 開啟瀏覽器到測試網址
3. 準備一組測試用會員電話（有點數的）

---

## 1. 🔐 登入閘門

| 步驟 | 預期結果 |
|------|---------|
| 開啟 POS 首頁 | 顯示 Google 登入按鈕 |
| 點擊 Google 登入 | 跳出 Google 帳號選擇 |
| 選擇白名單帳號登入 | 進入 POS 主畫面，顯示帳號名稱 |
| 嘗試非白名單帳號 | 顯示「此帳號無權限使用」紅色 banner |

---

## 2. 🛒 結帳流程

### 2a. 查詢會員
| 步驟 | 預期結果 |
|------|---------|
| 在電話欄輸入會員電話 | 出現自動完成下拉選單 |
| 按 Enter 查詢 | 顯示「查到 XXX，持有 N 點」綠色 banner |
| 輸入不存在的電話 | 顯示「找不到會員」紅色 banner |

### 2b. 福袋結帳（一般）
| 步驟 | 預期結果 |
|------|---------|
| 輸入套號 | 自動帶入套名、單價、獎項 |
| 輸入抽數 | 金額 = 單價 × 抽數，超過庫存量不可輸入 |
| 輸入現金/刷卡/匯款 | 即時計算應付 vs 實收 |
| 點擊結帳 | 成功 banner + 畫面清空 + 會員點數更新 |

### 2c. 點數套結帳
| 步驟 | 預期結果 |
|------|---------|
| 輸入點數套的套號 | type 自動變「點數」、金額歸零 |
| 顯示紅色「扣抵點數」欄位 | 預設 = 單抽價/20 × 抽數 |
| 結帳 | 顯示新點數（應減少）|

### 2d. 防呆驗證
| 步驟 | 預期結果 |
|------|---------|
| 空單結帳 | 紅色 banner 提示 |
| 未輸入電話 | 紅色 banner 提示 |
| 未查詢會員就結帳 | 紅色 banner 提示 |
| 金額為負數 | 紅色 banner 提示 |
| 點數不足 | 紅色 banner：「客戶點數不足」|
| 連點結帳（雙擊） | 第二次被攔截（isSubmitting 防護）|

---

## 3. 📋 當日銷售 Tab

### 3a. 資料顯示
| 步驟 | 預期結果 |
|------|---------|
| 切到「當日銷售」tab | 載入並顯示今日所有交易 |
| 切換「精簡對帳 / 經典模式」 | UI 正確切換 |
| 精簡對帳模式 | 顯示福袋編號（#lotteryId）|
| 統計區 | 正確顯示總營收、現金、刷卡、匯款、點數 |
| 純點數交易 | footer 顯示紫色「純點數交易」badge |
| 點數異動 | footer 正確顯示點數異動值（含 K 欄 fallback）|

### 3b. 開櫃準備金
| 步驟 | 預期結果 |
|------|---------|
| 輸入準備金金額 | 輸入時不會觸發 API（只在 blur 才寫入）|
| 點其他地方（blur）| banner 顯示「設定成功」|
| 重新整理頁面 | 準備金保留（從 GAS 讀回）|
| 切換門市 | 顯示該門市的準備金（分開存）|

---

## 4. ❌ 作廢訂單

| 步驟 | 預期結果 |
|------|---------|
| 點擊訂單的「作廢」按鈕 | 顯示作廢確認 modal |
| Modal 顯示 | 電話、時間、實收、商品數 |
| 有點數異動的訂單 | modal 顯示「點數異動 +N（作廢後將退回）」|
| 確認作廢 | 訂單消失 + banner「已作廢且點數已退回」|
| 會員點數 | 正確退回（檢查會員資料 tab）|
| 跨分店作廢 | 被攔截（竹北不能廢金山的單）|

---

## 5. 🏪 門市切換

| 步驟 | 預期結果 |
|------|---------|
| 切換到另一門市 | 當日銷售刷新為該門市資料 |
| 獎項庫 | 只顯示該門市的獎項 |
| 庫存 | 刷新為該門市庫存 |
| 唯讀模式（非本店） | 結帳/作廢/關帳按鈕不可用 |

---

## 6. 🔒 關帳

| 步驟 | 預期結果 |
|------|---------|
| 非當天日期 | 關帳按鈕隱藏 |
| 點擊關帳 | 顯示關帳 modal（準備金、應收、實收）|
| 輸入實收金額 | 顯示差異金額 |
| 確認關帳 | 當日銷售清空 + 準備金歸零 |

---

## 7. 📦 獎項庫

| 步驟 | 預期結果 |
|------|---------|
| 切到獎項庫 tab | 顯示所有獎項 |
| 點數套獎項 | 顯示紫色「點數套」badge |
| 開套 | 填入貨號/名稱/抽數 → 成功入庫 |
| 作廢整套 | 彈出確認 → 成功刪除 |

---

## 8. 👥 會員資料

| 步驟 | 預期結果 |
|------|---------|
| 切到會員 tab | 顯示所有會員列表 |
| 搜尋會員 | 支援姓名/電話搜尋 |
| 點擊會員查看歷史 | 顯示該會員消費紀錄 |

---

## 9. 📱 客戶面頁面

| 步驟 | 預期結果 |
|------|---------|
| 前往 `/#/member` | 顯示會員登入頁 |
| 輸入電話+生日 | 查到：顯示遮罩姓名 + 點數 |
| 連續錯誤 5 次 | 顯示「嘗試次數過多，請 1 分鐘後再試」|
| 前往 `/#/stocklist` | 顯示兌換清單 |
| 前往 `/#/about` | 顯示關於頁 |

---

## 10. 🔄 Race Condition 驗證

| 步驟 | 預期結果 |
|------|---------|
| 開兩個分頁，同一會員 | 兩邊都查到同樣點數 |
| 兩邊幾乎同時結帳 | 兩筆都成功，點數加總正確 |
| 結帳 + 作廢同時操作 | 不衝突（ScriptLock 排隊）|
